Book Shop project

Project contains: BookShop.API - RESTful Web API and BookShop.WebApplication - MVC ASP.NET Core Web App.
To review this project on GitHub please follow this link.

BooksShop.API is an ASP.NET Core controller-based RESTful Web API for e-comerce Book Store website that uses Mongo Atlas shared Replica Set database for retriving products(books information) and local SQL (localdb)\MSSQLLocalDB Server for user accounts and orders. Requiers: internet connection and SQL Server (localdb)\MSSQLLocalDB with out password.

In this project used:
MongoDB Atlas Shared replica set for storing/retriving books collection and local SQL database for storing user accounts.

API Security:
API version headers, JWT tokens, API key and SQL database for Authorization and Authentication

Async methods in API, providing the ability to handle several concurrent HTTP requests. They are not blocking the main thread while waiting for the database response. This API consumes and produces data in Json format, because this format is simple and lightweighted.

Current API supports 3 API-Versions:

  1. First API-Version is for Admin access only.
  2. Second API-Version for User's access.
  3. Third API-Version for not Signed in users, for guests.

Mongo Atlas Shared Replica set contains all data about books library.

For API-Version:3, don't need to provide JWT token. But this versions only supports some GET methods from StockController.cs and POST methods from AuthenticationConroller.cs to register new account only.
The list of all methods and supporting API-Version for each method could be found at the end of this page.
API-Version:1 supports all methods of API (GET, POST, PUT, DELETE), for access to it requires: api-key and valid JWT token.
To abtain JWT token, required provide api-key and correct admin loging details, after thet API will generate JWT token that valid for 1 hour.
For default accounts the token sends via http request message, otherwise sends token to the email.

Method API:
GET/books/page
Description:
Returns a list of requested products for requested page
API Versions supported:
ApiVersion-BooksStore: 1
ApiVersion-BooksStore: 2
Additional notes:
Returns list of requested quantity of products for requested page.
The list is sorted in requested order, by requested parameter(by Title or by Author or by Price).
If not specified sorting order && parameter for order, then its uses default parameters: descending order by Price.
If requested page < 1, then returns page 1, if requested page is larger then total pages, then returns last page.
The requested quantity of products per page should be in range 5 to 30. If requested quantity < 5 returns 5 products, if > 30 returns 30.
For ApiVersion-BooksStore : 2 retrieves only were parameter “isAvailable” : true
